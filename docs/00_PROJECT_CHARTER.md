# 00｜Project Charter

## 1. 專案使命

以現代瀏覽器技術重新詮釋「守自己的塔、同時出怪攻擊對手」的雙向塔防樂趣，先打造一個規則清楚、可重播、可平衡、可延伸成線上競技的單機原型。

## 2. 玩家核心幻想

同一筆資源持續做三種選擇：

1. **現在活下來**：建造或升級塔。
2. **未來更有錢**：出怪提高永久 Income。
3. **現在壓垮對手**：選擇克制對方防線的怪物與時機。

## 3. 產品支柱

### P1｜雙向決策

防守和進攻共享經濟，任何選擇都有機會成本。

### P2｜可讀的克制

四塔四怪必須有明確角色，玩家能從速度、體型、護盾與群聚理解威脅。

### P3｜公平可重現

相同 Seed 與 Command Log 必須產生相同結果。AI 不作弊，未來 Server 不信任 Client 結果。

### P4｜短局但有成長曲線

目標對局 8–12 分鐘；前期建立基礎、中期辨識策略、後期由收入差與壓力決勝。

### P5｜先好玩，再擴張

任何 Roguelike、放置、Live Ops 或商業化功能，都不能先於核心對局驗證。

## 4. 目標玩家

- 喜歡塔防與資源管理。
- 喜歡 1v1 心理博弈但不要求高操作速度。
- 熟悉自訂地圖式對抗塔防。
- 願意參與小規模 PC 瀏覽器測試。

## 5. 目標平台

MVP：

- 桌面瀏覽器。
- 鍵盤＋滑鼠。
- 邏輯解析度 1280×720。
- 最低可用視窗 1024×576。

延後：

- 完整手機觸控 UX。
- 原生 App。
- 主機。

## 6. 核心成功定義

- 玩家理解守塔與出怪的經濟關係。
- 四塔四怪用途可辨識。
- AI 對局穩定完成。
- 至少 70% 測試局主動出怪。
- 至少 60% 測試者願意立即再玩。
- 完整局中位數 7–12 分鐘。
- 因規則不清造成的首局中斷率低於 15%。
- Replay 可完全重現結果。

## 7. 非目標

- 大量內容與完整美術。
- 手機商業化。
- 帳號、商城、廣告。
- 天梯、賽季、社交。
- 長期數值成長。
- 複雜物理與彈道。
- 程序地圖。

## 8. 原創風險

`Chaos TD` 是工程代號，不代表最終商業名稱。公開前須完成名稱、Logo、世界觀、角色與素材原創化及法律檢查。
